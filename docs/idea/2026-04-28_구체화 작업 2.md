정리하면 서비스의 중심을 이렇게 잡는 구조입니다.

> **주 타깃:** 관심 분야의 최신 트렌드만 골라 보고 싶은 사용자
> **보조 타깃:** 요즘 전체적으로 유행하는 이슈도 빠르게 확인하고 싶은 사용자
> **서비스 중심:** 관심사 기반 개인화 트렌드 탐색
> **보조 기능:** 전체 최신 이슈 탭

이렇게 잡으면 교수님 지적이었던 **“대상과 목적이 모호하다”**는 문제를 상당히 줄일 수 있습니다.

# 1. 현재 방향의 핵심 구조

지금 선택하신 방향은 단순히 “트렌드 앱”이 아니라,

> **내 관심사 기준으로 최신 흐름을 골라 보는 개인화 트렌드 앱**

에 가깝습니다.

여기서 중요한 점은 **최신 이슈 탭은 메인이 아니라 보조 기능**으로 두는 것입니다.

만약 발표에서 “최신 이슈도 보여주고, 관심사도 보여주고, AI 분석도 하고, 커뮤니티도 합니다”라고 말하면 다시 모호해집니다.

반대로 이렇게 말하면 훨씬 선명합니다.

> 본 서비스의 핵심은 사용자의 관심 분야를 기준으로 트렌드를 선별해 보여주는 것입니다.
> 다만 사용자가 전체적인 사회적 흐름도 빠르게 확인할 수 있도록, 보조적으로 최신 이슈 탭을 제공합니다.

즉, **관심사 기반 탭이 주인공**, **최신 이슈 탭은 전체 흐름 확인용 보조 화면**입니다.

---

# 2. 타깃 정의를 더 날카롭게 다듬기

현재 문장:

> 패션, 뷰티, 게임, 음식 등 자신의 관심 분야 최신 트렌드만 골라 보고 싶은 사용자

이 문장도 좋지만, 발표용으로는 조금 더 목적을 넣어주면 좋습니다.

## 개선 문장

> 본 서비스의 핵심 타깃은 패션, 뷰티, 게임, 음식 등 자신의 관심 분야에서 어떤 주제가 유행하고 있는지 빠르게 파악하고 싶은 사용자입니다.

조금 더 구체적으로 쓰면:

> 특히 여러 플랫폼을 직접 확인하지 않고도, 자신의 관심사에 맞는 최신 트렌드를 한 화면에서 확인하고 싶은 10~20대 모바일 사용자를 주요 대상으로 설정했습니다.

이렇게 하면 대상이 더 명확해집니다.

- 관심 분야가 있는 사람
- 최신 흐름을 빠르게 알고 싶은 사람
- 여러 플랫폼을 일일이 보기 귀찮은 사람
- 모바일 환경에서 짧게 확인하고 싶은 사람

이 네 가지가 자연스럽게 들어갑니다.

---

# 3. 서비스 목적 문장 예시

발표나 기능상세명세서 앞부분에는 이런 식으로 쓰면 좋습니다.

```text
본 서비스는 사용자가 자신의 관심 분야에 맞는 최신 트렌드를 빠르게 탐색할 수 있도록 돕는 개인화 트렌드 큐레이션 앱이다.

기존의 트렌드 정보는 플랫폼별로 흩어져 있고, 사용자가 관심 없는 이슈까지 함께 확인해야 하는 불편함이 있다. 이에 본 서비스는 사용자의 관심 카테고리를 기반으로 관련 트렌드를 우선 제공하고, 보조적으로 전체 최신 이슈 탭을 통해 현재 사회적으로 주목받는 주제도 함께 확인할 수 있도록 한다.
```

이 문장은 꽤 발표용으로 좋습니다.
왜냐하면 다음 요소가 들어가 있기 때문입니다.

- 서비스 정체성: 개인화 트렌드 큐레이션 앱
- 문제: 플랫폼별 분산, 관심 없는 이슈 노출
- 해결: 관심 카테고리 기반 제공
- 보조 기능: 전체 최신 이슈 탭

---

# 4. 앱 구조도 이렇게 잡으면 좋습니다

기능 구조를 발표할 때는 아래처럼 **주 기능과 부 기능을 분리**해서 보여주는 것이 좋습니다.

```text
[메인 구조]

1. 관심사 기반 트렌드 탭 - 핵심 기능
   - 사용자가 선택한 관심 카테고리 기반 트렌드 제공
   - 패션, 뷰티, 게임, 음식 등 카테고리별 필터링
   - AI 기반 요약/관련 키워드 제공

2. 최신 이슈 탭 - 보조 기능
   - 전체 사용자 기준 인기 이슈 제공
   - 오늘/어제/주간 기준 트렌드 확인
   - 관심사 외의 넓은 흐름 파악

3. 트렌드 상세 화면
   - 키워드 설명
   - 유행 이유
   - 관련 키워드
   - 관련 콘텐츠/링크
```

여기서 중요한 건 **최신 이슈 탭을 독립적인 메인 목표처럼 말하지 않는 것**입니다.

좋은 표현:

> 최신 이슈 탭은 개인 관심사 밖의 큰 흐름을 보완적으로 확인하기 위한 기능입니다.

아쉬운 표현:

> 저희 앱은 관심사 트렌드도 보여주고 최신 이슈도 보여줍니다.

두 문장은 비슷해 보이지만, 발표에서는 차이가 큽니다.
첫 번째는 설계 의도가 있고, 두 번째는 기능 나열처럼 들립니다.

# 5. 발표에서 설명할 수 있는 사용자 시나리오

이 방향은 사용자 시나리오로 설명하면 설득력이 좋아집니다.

## 예시 시나리오

```text
사용자는 처음 앱을 실행할 때 패션, 뷰티, 게임, 음식 등 관심 카테고리를 선택한다.

이후 메인 화면에서는 사용자의 관심 분야와 관련된 최신 트렌드가 우선적으로 노출된다.

예를 들어 게임을 관심사로 선택한 사용자는 최근 인기 게임, 게임 밈, 업데이트 이슈 등을 확인할 수 있고, 음식에 관심 있는 사용자는 최근 유행하는 메뉴나 맛집 관련 키워드를 볼 수 있다.

또한 최신 이슈 탭에서는 개인 관심사와 별개로 현재 전체적으로 주목받는 트렌드를 확인할 수 있어, 사용자는 개인화된 정보와 대중적인 흐름을 함께 파악할 수 있다.
```

이런 식으로 말하면
“왜 관심사 탭이 있고, 왜 최신 이슈 탭이 필요한지”가 자연스럽게 설명됩니다.

# 6. 기능상세명세서에 반영할 때의 기준

기능 명세서를 작성할 때는 우선순위를 이렇게 나누는 게 좋습니다.

## 1순위 기능: 반드시 있어야 함

```text
- 회원가입/로그인
- 관심 카테고리 선택
- 관심사 기반 트렌드 목록 조회
- 트렌드 상세 조회
- 최신 이슈 목록 조회
```

## 2순위 기능: 차별화 기능

```text
- AI 기반 트렌드 요약
- 관련 키워드 추천
- 관심사 수정
- 트렌드 저장/북마크
```

## 3순위 기능: 확장 기능

```text
- 커뮤니티
- 댓글
- 알림
- 친구/팔로우
- 개인 활동 기반 추천 고도화
```

캡스톤에서는 1순위와 2순위 일부만 제대로 구현해도 충분히 설득력 있습니다.
특히 커뮤니티까지 초기에 넣으면 서비스 목적이 다시 흐려질 수 있습니다.

# 7. 이번 지적에 대한 보완 답변 예시

교수님이나 팀 회의에서 이렇게 답변할 수 있습니다.

```text
기존에는 ‘최신 트렌드 제공’이라는 표현이 넓어서 대상과 목적이 모호했습니다.

이를 보완하기 위해 핵심 대상을 ‘자신의 관심 분야 최신 트렌드를 빠르게 확인하고 싶은 사용자’로 좁혔습니다.

서비스의 중심 기능은 사용자가 선택한 관심 카테고리를 기반으로 트렌드를 선별해 보여주는 것이며, 최신 이슈 탭은 전체적인 흐름을 함께 파악하기 위한 보조 기능으로 정리했습니다.

따라서 본 서비스는 단순한 인기 검색어 조회 앱이 아니라, 관심사 기반 개인화 트렌드 큐레이션 앱을 목표로 합니다.
```

이 답변은 상당히 깔끔합니다.
지적을 수용했고, 방향도 재정의했고, 기능 우선순위도 정리되어 있습니다.

# 8. 생각해 볼 점

여기서 추가로 팀에서 정해야 할 것이 있습니다.

## A. 관심사는 사용자가 직접 선택하는가, AI가 추론하는가?

초기 버전에서는 직접 선택이 좋습니다.

```text
초기 버전:
사용자가 관심 카테고리를 직접 선택

확장 버전:
사용자의 클릭, 저장, 검색 기록을 기반으로 AI가 관심사를 보정
```

처음부터 AI가 모든 관심사를 자동 분석한다고 하면 구현 난도가 올라가고 설명도 어려워집니다.

---

## B. 카테고리는 몇 개로 시작할 것인가?

처음에는 4~6개 정도가 적당합니다.

예:

```text
패션
뷰티
게임
음식
엔터테인먼트
IT/디지털
```

너무 많으면 데이터 수집과 UI가 복잡해집니다.

---

## C. 최신 이슈는 어떤 기준으로 보여줄 것인가?

이것도 명확히 해야 합니다.

예:

```text
최신 이슈 탭은 사용자의 관심사와 무관하게, 전체 인기 키워드를 일간/주간 기준으로 제공한다.
```

또는

```text
최신 이슈 탭은 플랫폼별 인기 키워드를 종합하여 현재 주목받는 이슈를 보여준다.
```

이 기준이 없으면 “최신”이라는 단어가 다시 모호해집니다.

# 최종 정리

현재 방향은 이렇게 정리하면 좋습니다.

```text
우리 서비스는 ‘최신 트렌드 전체를 보여주는 앱’이 아니라,
‘사용자 관심사에 맞는 최신 트렌드를 선별해 보여주는 개인화 트렌드 큐레이션 앱’이다.

핵심 타깃은 패션, 뷰티, 게임, 음식 등 자신의 관심 분야에서 유행하는 주제를 빠르게 확인하고 싶은 사용자이다.

최신 이슈 탭은 서비스의 주 기능이 아니라, 개인 관심사 밖의 전체 흐름을 보조적으로 확인하기 위한 기능으로 둔다.
```

이제 기획 방향은 꽤 좋아졌습니다.
다음 단계에서는 **“관심사 기반 트렌드 탭이 메인 화면에서 어떻게 보여야 하는가”**를 UI/기능명세 기준으로 구체화하면 됩니다.
