# Trend Leader 최소 추천 순서도 8개 서술 정리

이 문서는 **Trend Leader** 캡스톤디자인 작품의 핵심 기능 중, 우선적으로 순서도로 그릴 만한 **최소 추천 세트 8개**를 말로 풀어 쓴 초안이다.  
목표는 단순히 박스를 나열하는 것이 아니라, **사용자 행동 → 시스템 처리 → 조건 분기 → 결과**의 흐름을 자연스럽게 정리하여 실제 순서도 작성의 출발점으로 삼는 데 있다.

---

## 1. 앱 시작 / 로그인 상태 분기

이 순서도는 사용자가 앱을 처음 실행했을 때 어떤 경로로 진입하는지를 설명한다.

사용자가 앱을 실행하면 먼저 스플래시 화면이 잠시 표시된다. 시스템은 이 시점에서 사용자의 로그인 상태를 확인한다. 예를 들어, 이전 로그인 정보나 토큰이 유효하게 저장되어 있는지를 검사한다.  
로그인 상태가 유효하다면 사용자는 별도의 인증 절차 없이 바로 홈 화면으로 이동한다. 반대로 로그인 정보가 없거나 만료되었다면 로그인/회원가입 화면으로 이동한다.

즉, 이 순서도는 다음과 같은 질문에 답한다.
- 사용자가 앱을 실행했을 때 바로 홈으로 갈 수 있는가?
- 아니면 인증이 필요한가?

핵심 분기점은 **"로그인 상태가 유효한가?"** 이다.

---

## 2. 로그인

이 순서도는 사용자가 계정으로 앱에 접속하는 과정을 설명한다.

사용자는 로그인 화면에 진입한 뒤, 일반 로그인 또는 소셜 로그인을 선택한다. 일반 로그인의 경우 아이디(또는 이메일)와 비밀번호를 입력한 뒤 로그인 요청을 보낸다. 시스템은 입력값이 올바른 형식인지 확인하고, 이후 회원 정보와 비밀번호가 일치하는지 검증한다.  
검증에 성공하면 로그인 처리가 완료되고 홈 화면으로 이동한다. 실패하면 오류 메시지를 보여주고 로그인 화면에서 다시 입력하도록 한다.

소셜 로그인의 경우에는 외부 인증 과정을 거친 뒤, 인증 결과를 바탕으로 기존 회원인지 확인한다. 기존 회원이면 바로 로그인 처리하고, 최초 로그인 사용자라면 추가 정보 입력 또는 회원 연동 절차로 이동시킬 수 있다.

핵심 분기점은 다음과 같다.
- 로그인 방식은 일반 로그인인가, 소셜 로그인인가?
- 입력 정보 또는 인증 결과가 유효한가?
- 기존 회원인가?

---

## 3. 회원가입(일반)

이 순서도는 일반 회원가입 절차를 설명한다.

사용자가 회원가입 화면으로 이동하면 먼저 이메일 또는 아이디를 입력한다. 시스템은 이미 존재하는 계정인지 중복 여부를 검사한다.  
중복된 계정이라면 가입을 진행할 수 없으므로 사용자에게 다시 입력하도록 안내한다. 중복이 아니라면 비밀번호를 입력받고, 비밀번호 길이, 문자 조합 등 기본 유효성 조건을 검사한다. 이후 이메일 인증 또는 인증코드 확인 절차가 있다면 인증코드를 발송하고, 사용자가 올바른 코드를 입력했는지 확인한다.

인증이 완료되면 닉네임 등 추가 정보를 입력받는다. 시스템은 닉네임 중복 여부도 검사한다. 닉네임까지 문제없다면 회원가입을 완료하고, 가입 완료 메시지를 보여준 뒤 로그인 화면 또는 홈 화면으로 이동시킨다.

핵심 분기점은 다음과 같다.
- 이메일/아이디가 중복되는가?
- 비밀번호가 유효한가?
- 인증코드가 정확한가?
- 닉네임이 중복되는가?

이 기능은 입력, 검증, 조건 분기, 완료 처리까지 모두 들어 있으므로 대표적인 순서도 대상이다.

---

## 4. 트렌드 리스트 조회

이 순서도는 사용자가 다양한 플랫폼의 트렌드를 조건별로 조회하는 과정을 설명한다.

사용자가 트렌드 리스트 화면에 진입하면 시스템은 기본 조회 조건을 설정한다. 예를 들어 기본 플랫폼, 기본 날짜, 전체 카테고리 등이 초기값으로 잡힐 수 있다. 사용자는 이후 플랫폼 탭을 선택하고, 날짜 필터를 고르며, 카테고리 필터를 선택한다.  
시스템은 사용자가 선택한 조건을 바탕으로 트렌드 데이터를 조회한다. 조회 결과가 존재하면 순위 리스트 형태로 화면에 출력한다. 결과가 없다면 해당 조건에 맞는 데이터가 없음을 안내한다.

이후 사용자는 특정 키워드를 선택하여 상세 화면으로 이동할 수 있다.

핵심 분기점은 다음과 같다.
- 어떤 플랫폼을 선택했는가?
- 어떤 날짜와 카테고리를 선택했는가?
- 조회 결과가 존재하는가?

이 순서도는 서비스의 핵심 데이터 탐색 기능을 보여주는 중요한 흐름이다.

---

## 5. 트렌드 상세 조회 및 저장

이 순서도는 사용자가 특정 트렌드 키워드를 눌렀을 때 상세 정보를 확인하고 저장하는 과정을 설명한다.

사용자가 트렌드 리스트에서 특정 키워드를 선택하면 시스템은 해당 키워드에 대한 상세 정보를 조회한다. 여기에는 키워드 설명, 순위 변화, 관련 키워드, AI 분석 결과, 관련 게시글 등이 포함될 수 있다.  
각 정보가 준비되면 상세 화면에 통합적으로 표시된다. 이때 사용자는 저장 버튼을 누를 수 있다. 저장 버튼을 눌렀을 경우 시스템은 먼저 로그인 여부를 확인한다. 로그인하지 않은 사용자라면 로그인 화면으로 이동시키거나 로그인이 필요하다는 메시지를 표시한다.

로그인한 사용자라면 이미 저장한 키워드인지 확인한다. 저장되지 않은 상태라면 저장 처리하고, 이미 저장된 상태라면 저장 해제 처리할 수도 있다. 이후 저장 상태를 화면에 반영한다.

핵심 분기점은 다음과 같다.
- 상세 데이터 조회가 가능한가?
- 로그인한 상태인가?
- 이미 저장된 키워드인가?

이 순서도는 단순 조회 기능을 넘어, 개인화 기능과 상태 변화까지 함께 담고 있다.

---

## 6. AI 추천 생성

이 순서도는 사용자의 관심사를 바탕으로 추천 트렌드를 생성하는 과정을 설명한다.

사용자가 추천 화면에 진입하면 시스템은 먼저 추천에 활용할 수 있는 사용자 데이터를 확인한다. 예를 들어 사용자가 설정한 관심 카테고리, 저장한 트렌드, 최근 본 키워드 등이 존재하는지를 검사한다.  
이 정보가 충분하다면 추천 기준을 정하고, AI 또는 추천 로직에 필요한 데이터를 구성한다. 이후 추천 요청을 수행한 뒤 결과를 받아 화면에 카드 형태 등으로 출력한다.

반대로 사용자 데이터가 부족하다면, 기본 인기 트렌드를 기반으로 추천하거나 관심 카테고리 설정을 먼저 유도할 수 있다. 추천 결과가 없을 경우에는 대체 추천 리스트를 보여주는 방식도 가능하다.

핵심 분기점은 다음과 같다.
- 추천에 활용할 사용자 정보가 충분한가?
- 관심사 기반 추천을 할 것인가, 저장 키워드 기반 추천을 할 것인가?
- 추천 결과가 존재하는가?

이 기능은 Trend Leader의 차별성을 보여주는 대표 흐름이므로 순서도로 그릴 가치가 크다.

---

## 7. 게시글 작성

이 순서도는 사용자가 커뮤니티에 글을 등록하는 과정을 설명한다.

사용자가 게시판에서 글쓰기 버튼을 누르면 시스템은 먼저 로그인 여부를 확인한다. 로그인하지 않은 사용자는 글을 작성할 수 없으므로 로그인 화면으로 이동시키거나 로그인이 필요하다는 안내를 보여준다.  
로그인한 사용자라면 글쓰기 화면으로 이동하여 제목, 본문, 카테고리, 필요하다면 관련 트렌드 키워드까지 입력할 수 있다.

사용자가 작성 완료 후 등록 버튼을 누르면 시스템은 입력값 유효성을 검사한다. 제목이나 본문이 비어 있으면 다시 입력하도록 안내한다. 문제가 없다면 게시글을 저장하고 등록 완료 메시지를 보여준다. 이후 게시글 상세 페이지나 게시판 목록으로 이동한다.

핵심 분기점은 다음과 같다.
- 로그인한 상태인가?
- 제목과 본문이 유효한가?
- 관련 트렌드를 연결할 것인가?
- 등록이 성공했는가?

이 흐름은 게시판 기능의 가장 대표적인 생성(Create) 흐름이다.

---

## 8. 게시글 상세(댓글/좋아요 포함)

이 순서도는 사용자가 게시글 상세 화면에서 본문을 읽고 상호작용하는 과정을 설명한다.

사용자가 게시글 목록에서 특정 글을 선택하면 시스템은 게시글 본문과 작성자 정보, 댓글 목록, 좋아요 수, 관련 트렌드 링크 등을 조회하여 화면에 출력한다.  
이후 사용자는 여러 행동을 할 수 있다. 댓글을 작성하려고 하면 시스템은 로그인 여부를 확인하고, 로그인한 사용자라면 댓글 입력 후 등록 처리한다. 좋아요 버튼을 누를 경우에도 로그인 여부를 확인한 뒤, 아직 좋아요를 누르지 않았다면 좋아요를 추가하고, 이미 누른 상태라면 좋아요를 취소할 수 있다.

관련 트렌드 링크를 누르면 해당 트렌드 상세 화면으로 이동할 수도 있다.

핵심 분기점은 다음과 같다.
- 게시글 상세 데이터가 정상적으로 조회되는가?
- 사용자가 댓글을 작성하는가, 좋아요를 누르는가, 관련 트렌드로 이동하는가?
- 로그인한 상태인가?
- 이미 좋아요를 누른 상태인가?

이 순서도는 단순 조회와 사용자 상호작용이 결합된 대표 사례다.

# 마무리 정리

위 8개는 Trend Leader에서 가장 먼저 순서도로 그려볼 만한 최소 핵심 세트이다.

정리하면 다음과 같다.

1. 앱 시작 / 로그인 상태 분기  
2. 로그인  
3. 회원가입(일반)  
4. 트렌드 리스트 조회  
5. 트렌드 상세 조회 및 저장  
6. AI 추천 생성  
7. 게시글 작성  
8. 게시글 상세(댓글/좋아요 포함)  

이 8개를 먼저 정리하면, 이후에는 자연스럽게 다음 기능들도 확장할 수 있다.
- 소셜 회원가입
- 게시글 목록/검색
- 관심 카테고리 설정
- 저장한 트렌드 조회
- 내가 쓴 글 관리

즉, 지금 단계에서는 **모든 기능을 한 번에 다 그리려고 하기보다**, 먼저 핵심 사용자 흐름 8개를 말로 구조화한 뒤 하나씩 순서도로 옮기는 방식이 가장 안정적이다.  
순서도는 처음부터 예쁘게 그리려 하기보다, 먼저 이 문서처럼 **문장으로 흐름을 정리하고**, 그다음에 **시작 → 처리 → 판단 → 결과** 형태로 박스를 옮겨 적는 식으로 접근하면 훨씬 수월해진다.
