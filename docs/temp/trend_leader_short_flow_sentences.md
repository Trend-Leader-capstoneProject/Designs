# Trend Leader 순서도용 짧은 문장 정리

## 1. 게시글 작성

- 게시판 화면에 진입한다.
- 사용자가 글쓰기 버튼을 누른다.
- 로그인 상태인지 확인한다.
- 로그인이 되어 있지 않으면 로그인 화면으로 이동한다.
- 로그인이 되어 있으면 글쓰기 화면으로 이동한다.
- 제목, 내용, 관련 트렌드 태그를 입력한다.
- 필수 입력값이 비어 있는지 검사한다.
- 비어 있으면 다시 입력하도록 안내한다.
- 입력이 완료되면 게시글 등록을 요청한다.
- 등록이 성공하면 게시글 상세 화면으로 이동한다.
- 등록이 실패하면 오류 메시지를 출력한다.

---

## 2. 게시글 상세(댓글, 좋아요)

- 사용자가 게시글을 선택한다.
- 시스템이 게시글 상세 정보를 조회한다.
- 본문, 작성자, 작성일, 댓글 목록을 화면에 표시한다.
- 사용자가 좋아요 버튼을 누른다.
- 로그인 상태인지 확인한다.
- 로그인이 되어 있지 않으면 로그인 화면으로 이동한다.
- 로그인이 되어 있으면 좋아요 상태를 변경한다.
- 변경된 좋아요 수를 다시 표시한다.
- 사용자가 댓글 입력 후 등록 버튼을 누른다.
- 댓글 내용이 비어 있는지 확인한다.
- 비어 있으면 다시 입력하도록 안내한다.
- 비어 있지 않으면 댓글을 저장한다.
- 댓글 저장이 완료되면 댓글 목록을 새로고침한다.

---

## 3. 북마크

- 사용자가 트렌드 상세 또는 게시글 화면에 진입한다.
- 북마크 버튼을 누른다.
- 로그인 상태인지 확인한다.
- 로그인이 되어 있지 않으면 로그인 화면으로 이동한다.
- 로그인이 되어 있으면 기존 북마크 여부를 확인한다.
- 이미 북마크가 되어 있으면 북마크를 해제한다.
- 북마크가 되어 있지 않으면 북마크를 저장한다.
- 처리 결과를 화면에 반영한다.
- 사용자가 마이페이지에서 북마크 목록을 확인한다.
- 북마크 목록이 있으면 리스트를 출력한다.
- 북마크 목록이 없으면 저장된 항목이 없다고 안내한다.
